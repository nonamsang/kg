# 🍽 감정기반 굿즈 쇼핑몰 프로그램 (Emotion-Based Merchandise Shopping Mall Program)

| 구분 | 내용 |
|------|------|
| **교육기관** | KGITBANK |
| **과정명** | 핀테크 개발을 위한 풀스택 개발자 양성과정 |
| **교육기간** | 2025.03.10 ~ 2025.09.09 |
| **프로젝트명** | 감정기반 굿즈 쇼핑몰 프로그램 (Emotion-Based Merchandise Shopping Mall Program) |
| **개발기간** | 2025.06.09 ~ 2025.07.01 (총 22일) |
| **참여인원** | 5명 |
| **담당업무** | 사용자 리뷰 및 리뷰 댓글 시스템 구현 (100%)<br>사용자 주문 관리 시스템 구현 (50%)<br>관리자 상품 관리 시스템 구현 (100%) |
| **개발환경** | SPRING REGACY, Oracle 18c |
| **사용기술** | HTML, CSS, JAVASCRIPT, AJAX, SPRING(MVC모델 활용), MY BATIS, JSON |
| **사용 API** | KAKAO PAY 결제 API |
| **관련 링크** | https://blog.naver.com/pr1010327/224025617664 |

---

### 📘 개요
- 사용자의 감정을 분석하여 <b>맞춤형 상품</b>을 추천하고 개인화된 쇼핑 경험 제공
- 관리자 시스템을 통해 <b>상품, 주문, 리뷰</b>를 효율적으로 관리
- <b>Kakao Pay</b> 연동으로 안전하고 편리한 결제 환경 제공

### 👥 담당 역할
- **관리자 상품 관리**
  - 상품 등록, 수정, 삭제 기능 구현
  - 상품 등록 완료 시 옵션 등록, 수정, 삭제 가능하도록 설계
  - 사용자가 보기 편한 UI로 상품 옵션 구현
  - 상품 삭제 시 해당 옵션도 함께 삭제되도록 트랜잭션 활용
- **사용자 리뷰 관리**
  - 리뷰 등록, 수정, 삭제 기능 구현
  - 리뷰 등록 시 댓글 등록, 수정, 삭제 가능하도록 설계
  - 세션 기반 조건문 활용: 로그인한 사용자ID와 리뷰 작성자ID 일치 시만 수정/삭제 가능
  - 리뷰 삭제 시 댓글 포함 전체 삭제 처리 트랜잭션 활용

- **주문 내역 관리**
  - 주문 상세 조회, 배송 상태 조회 기능 구현
  - 가상 배송 상태: 하루 단위로 배송 상태 변경 구현
  - 배송 완료 시 리뷰 등록 기능 활성화

### 👥 프로그램 설계

- **사용자 주문 관리**
  - 사용자 마이페이지에서 주문 내역 전체 조회
  - 주문 상세 & 배송 상태 조회
  - 배송 완료 시 리뷰 등록 버튼 활성화
  - 리뷰 등록 시 리뷰 내용과 이미지 등록 기능

- **사용자 리뷰 관리**
  - 사용자 마이페이지에서 작성한 리뷰 전체 조회
  - 리뷰 작성자 ID와 세션 접속 ID 일치 시 리뷰 수정 & 삭제 버튼 활성화
  - 리뷰 수정 시 내용 및 이미지 수정, 리뷰 삭제 시 ALERT 확인 후 삭제

- **사용자 리뷰 댓글**
  - 상품 조회 → 리뷰 공개 → 리뷰 선택 → 댓글 조회 및 등록 기능 활성화
  - 댓글 작성자 ID와 세션 접속 ID 일치 시 댓글 수정 & 삭제 버튼 활성화

- **관리자 상품 등록**
  - 관리자 페이지에서 상품 등록 (상품 이름, 가격, 감정, 카테고리, 이미지 입력)

- **관리자 상품 조회**
  - 등록한 상품 전체 조회
  - CHECK BOX 활용하여 수정 & 삭제할 상품 선택 가능

- **관리자 상품 옵션 관리**
  - 관리자 페이지 → 상품 옵션 조회
  - RADIO 버튼으로 상품 선택 후 해당 상품의 옵션 전체 조회

- **상품 옵션 등록 (색상, 사이즈, 재고, 이미지 입력)**
  - 옵션 하나 선택 시만 수정 & 삭제 기능 활성화

### 🎯 구현 목표 요약

- ✅ 회원 관리 및 인증: USER_TBL과 MANAGER를 활용해 회원과 관리자 계정 정보 안전하게 관리  
- ✅ 업체 및 상품 관리: COMPANY1, PRODUCT, PRODUCT_OPTION 테이블 기반으로 입점 업체와 상품 정보 효율적으로 관리  
- ✅ 주문 및 장바구니 관리: ORDER_TBL, ORDER_DETAIL, BASKET, BASKET_DETAIL, WISHLIST, RECENT_VIEW_TBL 테이블을 활용해 주문, 장바구니, 관심 상품, 최근 본 상품 기능 
- ✅ 리뷰 및 문의 관리: REVIEW, REVIEW_SUB, QUESTION, ANSWER 테이블을 통해 리뷰, 대댓글, 상품 문의 및 답변 기능 제공  
- ✅ 공지/이벤트 관리: NOTICE_TBL, EVENT_TBL 테이블을 이용하여 공지사항과 이벤트/프로모션 정보를 사용자에게 전달  
- ✅ GUI 기반 편리한 예약/주문 인터페이스 제공: 사용자가 쉽게 상품을 선택, 주문, 리뷰 작성, 문의 가능  
- ✅ 시스템 안정성 및 데이터 무결성 확보: 테이블 구조를 기반으로 주문, 결제, 장바구니 등 핵심 기능 오류 최소화

### 🛠️ 프로젝트 후기

#### <기술적 측면>
- 첫 번째 프로젝트보다 난이도가 높았음. 감정 기반 쇼핑몰 '무드샵'에서는 최근 본 상품 기능 구현이 어려움.  
- 상품, 사용자, 최근본 상품 테이블을 조인하거나 서브쿼리를 작성해야 하는 복잡한 DB 처리 필요.  
- AJAX와 JavaScript를 활용해 사용자 행동에 따라 동적으로 최근 본 상품이 표시되도록 구현.  
- 트랜잭션 기능을 활용하여 이미 존재하는 더미 데이터를 삭제하고 최신 상품 정보를 저장하도록 처리.  
- 이를 통해 하나의 테이블에 필요한 컬럼이 모두 존재하지 않고, 다양한 테이블을 조합해야 기능 구현이 가능함을 깨달음.

#### <협업적 측면>
- 프로젝트 기능이 많아 혼자서는 힘들었으나, 팀원 간 적절한 분업으로 프로젝트 진행 가능.  
- 각자 맡은 기능을 구현하고, 서로의 구현 방식을 공유하며 문제 해결에 도움을 주고받음.  
- 구글 검색과 ChatGPT를 활용하여 어려운 구현 방법을 조사하고 팀 내 지식을 공유함으로써 효율적으로 문제를 해결함.

#### <개선적 측면>
- 기술적으로 어려운 기능도 포기하지 않고 지속적으로 학습하며 구현할 수 있는 역량 향상.  
- 향후 프로젝트에서는 DB 설계 시, 조인과 서브쿼리 구조를 미리 고려하여 설계 효율성을 높일 수 있음.  
- 팀 단위 프로젝트에서 기능 분업과 커뮤니케이션의 중요성을 다시 한번 인식, 협업 프로세스 개선 필요성 인지.
