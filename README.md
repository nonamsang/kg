### 🌱 개요
- 사용자가 환경인증 기업에 등록된 상품을 결제하면 해당 상품의 **탄소 배출량**을 계산하여 **에코 포인트** 적립  
- 적립된 에코 포인트를 통해 **개인의 친환경 소비 활동** 시각화 및 관리  
- 환경인증 기업의 판매 데이터와 탄소 절감 효과를 분석하여 **기업 인센티브** 제공

---

| **탄소 배출 인식** | 2026년부터 금융회사까지 온실가스 배출량("금융배출량") 공시 시대 도래, 기업과 개인이 탄소 배출을 직관적으로 인지 |
| **개인 맞춤 탄소 관리** | 카드 사용 내역 기반 **탄소 가계부**와 **그린 포인트 보상 시스템** 제공으로 친환경 소비와 저탄소 생활 습관 유도 |
| **기업 ESG 활동 지원** | 기업의 친환경 활동과 ESG 기반 투자 공시 및 홍보 채널 제공 |
| **환경적 지속가능성 강화** | 개인과 기업 모두가 탄소중립 달성에 기여할 수 있는 디지털 기반 그린핀테크 플랫폼 구현 |

- 디지털 기술 발전을 활용하여 개인과 기업 모두 **탄소 감축 효과**를 실생활에서 직관적으로 체감 가능  
- 금융 생태계와 대중 간의 **투명한 연결**을 통해 친환경 소비와 저탄소 전략 실행을 동시에 지원

# 🌱 프로젝트 목표: 환경 친화적 데이터 기반 시스템 구현

### 1. 사용자 경험 강화
- 마이페이지에서 **프로필, 활동 내역, 포인트, 결제 내역** 등 시각화 제공  
- **출석, 기부, 구매, 광고 시청** 등 활동에 따른 포인트 보상 시스템 구축  

### 2. 데이터 기반 탄소 감축 관리
- 결제 내역을 **CO₂ 절감량**으로 환산하고, **NetZero Point API** 연동  
- **Chart.js** 기반 월별·연간 탄소 감축량 시각화 제공  
- 가상 나무 성장 시스템으로 **포인트 적립과 친환경 행동 연동**  

### 3. 커뮤니티 및 소셜 기능
- 사용자 간 정보 공유, **댓글/좋아요 기능** 구현  
- **Cloudinary API** 활용 즉시 이미지 업로드 반영  

### 4. 관리자 / 상품 관리 기능
- 회원, 게시판, 채팅, 통계 관리  
- 상품 **추가·수정·삭제 기능**과 사진 관리  
- 실시간 서비스 및 확장성 구현  

### 5. 실시간 채팅 (WebSocket)
- 관리자와 사용자가 **실시간으로 대화**할 수 있는 시스템 구현

# 🛠 담당 기능 및 구현 상세

## 🌳 나무키우기 시스템
- **물주기 기능**
  - 물 재고가 있을 경우에만 작동, 재고 없으면 불가  
  - 비료 준 시간을 `sysdate`로 저장, 72시간 내 주었을 경우 경험치 +2, 그 외 +1  
  - 오픈소스 JSON 애니메이션 활용, 실제 물주는 느낌 구현  

- **비료주기 기능**
  - 비료 재고가 있을 경우만 작동, 없으면 불가  
  - 기존 비료 시간 즉시 최신으로 업데이트, 72시간 기준 타이머 적용  
  - **anime.js** 활용, 비료 주는 애니메이션 구현  

- **비료 구매 기능**
  - 사용자 포인트, 비료 재고, 포인트 내역 3개 테이블 트랜잭션 처리  
  - 포인트 부족 시 결제 불가, 충분 시 정상 결제 및 데이터 업데이트  

- **랜덤 잡초 이벤트**
  - `random()` 메소드로 발생 확률 지정  
  - CSS 활용, 잡초가 떠다니는 느낌 구현  
  - 광고보기 또는 포인트로 제거, 포인트 부족 시 광고만 가능 (`window.open()` 사용)  

- **레벨 7 달성**
  - 레벨 7 달성 시 물주기/비료주기 기능 비활성화  
  - 나무 다시 키우기 버튼 클릭 시 기존 나무 더미데이터 삭제, 재고 테이블 업데이트  
  - 완성된 나무는 **다 키운 나무 테이블**에 더미데이터 등록  

## 💬 커뮤니티 페이지
- CRUD 기능 구현  
- 이미지 업로드: **Cloudinary API** 활용, 성공 시 URL DB 저장  
- 좋아요 기능
  - 이미 눌렀으면 1 감소, 안 눌렀으면 1 증가  
  - 좋아요 기록 테이블 생성, 트랜잭션 기반 IF문으로 처리  
  - 비동기 방식으로 구현  

## 🧑‍💻 마이페이지 - 회원정보 수정
- 프로필 이미지: **Cloudinary API** 연동, 업로드 시 화면 반영  
- 닉네임: 중복 확인 후 수정 가능  
- 전화번호: 자동 하이픈 기능 적용  
- 주소: **다음API** 연동, 검색 및 선택으로 입력 가능  
- 이메일: **JavaMailSender** 활용, 인증 메일 발송 가능  

## 📨 문의 페이지
- 사용자 CRUD 문의 기능 구현  
- 관리자 답변 기능 구현  
- 답변 시 대댓글 기능 지원, 사용자와 관리자 간 원활한 커뮤니케이션 가능

# 🗂 주요 테이블 설명

---

## 1️⃣ 사용자 관련 테이블
- **user_table**: 회원 기본 정보 관리 (ID, 이름, 전화, 이메일, 포인트, 프로필 등)  
- **wallet_table**: 사용자의 지갑 정보 관리 (계좌, 잔액, 은행, 마지막 업데이트)  
- **point_table**: 사용자의 포인트 적립/사용 내역 기록  
- **point_event_table**: 특정 이벤트에 따른 포인트 지급 내역  
- **point_rule_table**: 포인트 지급 규칙 및 CO₂ 절감 단위 정의  

---

## 2️⃣ 상품 및 거래 관련 테이블
- **product_table**: 판매 상품 정보 관리 (이름, 카테고리, 가격, 탄소 배출량 등)  
- **payment_table**: 사용자의 결제 내역 기록  
- **donation_table**: 사용자 기부 내역 기록  
- **merchant_table**: 판매자 정보 관리 (상점 이름, 주소, 카테고리 등)  

---

## 3️⃣ 나무/환경 관련 테이블
- **growing_tree_table**: 사용자가 키우는 나무 정보 관리 (레벨, 이름, 물/비료 상태 등)  
- **grown_tree_table**: 완료된 나무 정보 기록 (완성된 나무, 저장된 물/비료 등)  
- **emission_table**: 사용자별 탄소 배출량 기록 (결제, 활동 기반)  
- **stock_table**: 나무 성장에 필요한 물/비료 재고 관리

# 🌟 프로젝트 경험 및 성장

## 1️⃣ 기술적 측면
- 주문 관리 및 리뷰 관리 시스템 구현을 통해 단일 기능이 다른 시스템과 어떻게 연동되는지 이해  
- 데이터 흐름 분석 능력 향상, 이미지 다중 업로드 문제 해결 과정에서 **트랜잭션 처리**와 **외래키 제약 조건** 적용 경험  
- 포인트 적립, 탄소 감축량 시각화, 가상 나무 성장 시스템 구현을 통해 **사용자 경험과 동기 부여**를 고려한 기능 설계 경험  
- 커뮤니티와 실시간 채팅 구현 과정에서 **WebSocket 기반 서버 통신**과 API 활용 능력 강화  

## 2️⃣ 협업적 측면
- 기능 구현 시 다른 시스템과의 연동을 고려하며 팀 내 **데이터 구조 및 기능 통합** 이해  
- 다중 사용자 환경에서도 안정적 동작이 가능하도록 설계하며 팀과의 협업 및 코드 통합 경험  
- 다양한 API와 외부 도구(Cloudinary, Chart.js, JavaMailSender 등)를 활용하여 **팀 프로젝트에서 공통 모듈화 및 표준화** 경험  

## 3️⃣ 개선적 측면
- 문제 발생 시 단순한 기능 수정이 아닌 **근본 원인 분석**과 해결책 적용 경험  
- 오류와 한계 상황에서도 포기하지 않고 지속적으로 개선하며 문제 해결 능력 강화  
- 사용자 경험과 기능 설계를 연결하여, **데이터 기반 문제 해결과 UX 개선을 동시에 고민**  
- 새로운 기술 학습과 적용 경험을 통해 빠르게 적응하고 실질적 성과를 만들어내는 자신감 획득
